Web Storage

Armazenamento Local

Armazenamento local é uma forma de guardarmos informações no navegador do usuário. Essas informações podem ser recuperadas mais tarde, quando ele retornar ao nosso site.

Tipos de informações que podem ser guardadas no navegador:

  • Dados e de acesso, como o nome de usuário.
  • Estado da aplicação, como a última página que ele visitou
  • Informações de configuração do site, que personalizam a sua experiência
  • Dados para funcionamento offline

API Web Storage

A API(Application Programming Interface) Web Storage oferece uma implementação simples e potente para o armazenamento local.

Essa API é implementada no navegador por meio de dois objetos pertencentes ao objeto window:

  • window.localStorage - Armazenamento permanente dos dados.
  • window.sessionStorage - Armazenamento por uma sessão(mesma janela e mesma aba, inclusive durante recargas da página)

Especificação: W3.org Web Storage

Vantagens

  • Armazenamento persistente
  • Facilidade de uso, meio de pares e valores (como os objetos JS)
  • Suporte em todos os navegadores
  • Não há envios dos dados para o servidor, garantindo privacidade e segurança

Desvantagens

  • Limite de 2+ MB(depende do navegador)
  • Só armazena strings
  • Não possui recursos de busca, indexação, atualização, versionamento, etc.

Operações

Opções para o armazenamento de dados:

  • localStorage.getItem("chave")
  • localStorage.chave
  • localStorage["chave"]

Opções para a recuperação do valor associado a uma chave.

  • localStorage.getItem("chave")
  • localStorage.chave
  • localStorage["chave"]

Opção para a remoção do valor associado a uma chave:

  • localStorage.removeItem("chave")
  • delete localStorage.chave
  • delete localStorage["chave"]

Opção para a remoção de todos os dados:

  • localStorage.clear()
